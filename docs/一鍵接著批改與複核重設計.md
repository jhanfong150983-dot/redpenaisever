# 一鍵接著批改 + 複核流程重設計（UI/UX 定稿 spec）

> 定稿日期：2026-05-31。本文是 SSoT，實作前/中對照此文。

## 1. 解決的痛點

學生在不同時間上傳，同一份作業裡每位學生狀態不一致；老師目前要**手動勾選**誰跑什麼、待複核又只能**一份一份開 detail 改**，很慢。另外「crop 看不到答案 → 老師選無法辨識 → 卡成待複核 → 又要進 detail 改」是多餘迴圈。

## 2. 四種狀態（狀態總覽列全部顯示）

> 2026-06-01 更新：**移除「待複核」狀態**。Phase A 跑了但人工審查沒做完 = Phase A 未完成 = **未擷取**（已上傳）；唯有審查完成、finalAnswers 存檔（或乾淨卷無待確認題）才算擷取完成 → **待批改**。模型更乾淨，且 Phase B only 不再卡在尷尬的中間態。

標籤用系統 `CARD_STAGE_LABEL` 真實值（不要自創）。

| | CardStage | 系統標籤 | 意思 | 下一步 |
|---|---|---|---|---|
| ⚪ | `not_submitted` | 未繳交 | 還沒上傳 | — |
| 🔵 | `not_extracted` | **未擷取** | 上傳了、Phase A 還沒跑 **或** 跑了但人工審查沒做完 | 智慧批改（Phase A + 審查） |
| 🟢 | `pending_grading` | **待批改** | Phase A + 審查完成（乾淨卷或審查存檔）、無待確認題 | 直接 Phase B |
| ✅ | `graded` | 已批改 | 全部完成 | — |

（另有 `phase_a_failed`=擷取失敗、`phase_b_failed`=批改失敗、`manual_marked`=手動標記已批改。）一鍵分桶：needA=未擷取（含「審查未完成」）+擷取失敗、needB=待批改+批改失敗。（needReview 已併入 needA。）

## 3. 按鈕策略（不取消、降級）

- **作業頁主按鈕：只有一顆「一鍵接著批改（N 份未完成）」**。
- **「進階 ⋯」選單**：放現在的「擷取答案（Phase A）」「批改作業（Phase B）」整批動作。
- **個別學生 detail 內**：保留「這份重跑 Phase A／重新批改」，針對單一學生用。
- 意圖區分：一鍵 = 前進（把沒做完的推到完成）；進階/detail 重跑 = 修正單一階段。

## 4. 勾選框只在進階模式出現

預設**無勾選框**（畫面清爽）。流程：按【進階】→ 選 Phase A 或 Phase B → 卡片進入勾選模式（出現 ☑ + 全選）→ 底部確認列「已選 N 份 ▶ 開始」→ 執行完自動退出勾選模式。

## 5. 一鍵接著批改流程

> 2026-06-01 實作更新：按鈕已改名 **「智慧批改」**（分段式，右側 ▼ = 進階選單）。實際 1c 排序為「**先對未擷取+審查未完成全跑 Phase A → 複核 → 對全 scope 跑一次統一 Phase B**」（非各自並行 Phase B），以確保最終總數正確且乾淨卷不停在中途。下述為原始設計意圖，細節以實作為準。並無「待複核」狀態：審查沒做完的卷顯示為 🔵 未擷取。

1. 老師按【智慧批改（6 份）】。
2. **一定先跳確認**，含**墨水警告**：「將處理 6 份、消耗約 N 次批改額度（墨水），確定？」
3. 系統分流（**各自好了就各自完成、不等全部**）：
   - 🔵 → 背景跑 Phase A。
   - 跑完乾淨的（🟢 + 乾淨的🔵）→ **背景立刻 Phase B → ✅**（不等老師）。
   - 有待複核的（🟡 + 不乾淨的🔵）→ 收集，進複核模式。
4. **複核模式（方案 B：依序開 detail）**：開第 1 份 → 老師改 → 按「完成這份，下一份 →」→ **那份立刻算分** → 自動開下一份。進度「複核中 2/3」。
5. 全部複核完 → 跳**最終總結** modal：「6 份：✅6 完成、⚠️0 失敗」。

## 6. Loading / Result UI 改法

流程從「按一下→卡住轉圈→跳結果」（一次性批次）變成「按一下→旁邊進度→中間請你複核（不卡畫面）→全好跳總結」（引導式）。

- **取消會卡住的全螢幕 spinner**，改成**常駐不卡畫面的狀態列**：`🔄 處理中 2 ‧ 🟡 待你複核 3 ‧ ✅ 完成 1 [暫停]` + 進度條。
- **Result modal 只留給「最終總結」**；過程進度移到狀態列即時更新。

## 7. 複核 detail 的設計（重點）

crop 維持主角；原圖按需召喚（不一直擺左側，否則 crop 失去意義）。

- 複核 detail 列出「需要你確認」的題，每題：crop + **輸入框**（看得清就確認、看不清就打字＝`manual`、會保留）。
- **把「無法辨識」改成「🔍 看原圖」按鈕**（平常不出現、需要才按）。
- 「看原圖」→ 打開**該題所在那一頁的原圖**（單頁、非多頁），用**可自由縮放+拖曳的檢視器**（此處為「圖片放大標準＝簡易 overlay」的**正當例外**，因用途是「整頁找答案」而非瞄 crop）。
- 檢視器**預設顯示整頁** + 畫一個**紅虛框標「AI 原本切的位置（bbox）」**當線索（不自動放大到 bbox，避免 bbox 大歪時帶到錯地方）。老師再放大到真答案、打字。
- 效果：審查當下就解決、不再丟回待複核；bbox 歪掉有原圖兜底；「已審查又叫複核」消失。「無法辨識」只剩真鬼畫符的最後手段。

## 8. 暫停 / 繼續

狀態列與複核導覽都有【暫停】：已複核完的保留完成、未複核的退回 🟡 待複核。老師之後按【繼續（一鍵）】時，**才再跳出複核**、接著做。系統永遠只處理還沒完成的，「不一條龍」沒關係、隨停隨接。

## 9. 仍待確認 / 風險

- 複核 RWD：小螢幕複核 detail 排版（手機上下排）。
- 「看原圖」單頁切圖：多頁作業要正確取出該題所在頁的影像 + bbox 對該頁的座標。
- 墨水警告的「N 次」估算：需從待處理份數 × 各階段 call 數推。
