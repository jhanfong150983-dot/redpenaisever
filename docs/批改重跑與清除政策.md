# 批改流程 — 重跑/更換答案卷 的「保留 vs 清除」政策（SSoT）

> 這份文件是「老師重跑某階段或更換答案卷時，後續狀態該怎麼走」的單一事實來源。
> 任何階段的清除/重建邏輯要改，先更新這份。最後更新：2026-05-30。

## 1. 流程與每階段產物

```
0 答案卷  → answerKey（題型/答案/rubric/vjRubric/crop）          ← 老師可重抽/編輯
A 預批改  → phase_a_state（每題 AI 讀值 + 一致/送審）            ← 可重跑
審查      → final_answers（老師確認後的答案）                    ← 老師手動
B 正式批改→ grading_result（分數 + 逐題 details + mistakes[]）   ← 可重跑
成績      → submissions.score / scoreSource                      ← 可手動改分
訂正      → correction_question_items + assignment_student_state  ← 可多輪
            訂正完成 = assignment_student_state.status='correction_passed' = endpoint
```

## 2. 實際狀態欄位（核對過）

- **submissions**：`score` / `ai_score` / `score_source('ai'|'manual')` / `grading_result` / `phase_a_state` / `final_answers` / `graded_at` / `round`(0=原卷,≥1=訂正) / `parent_submission_id`
- **grading_result.mistakes[]**：`{id, question, reason}`（本身不存訂正/申訴狀態）
- **assignment_student_state.status**（每生每作業一列）：
  `graded`(無需訂正) / `correction_required`(待訂正) / `correction_in_progress`(訂正中) / `correction_pending_review`(訂正完但有申訴待審) / **`correction_passed`(訂正完成 = endpoint、不可重跑)** / `correction_failed`(超次數)
- **correction_question_items**（每生每作業每題每輪一列）：
  `status: open | disputed | resolved | skipped`、`attempt_no`、`mistake_reason`、`dispute_note`(申訴理由)、`dispute_rejected_at` / `dispute_rejection_note`(駁回稽核)
- **申訴 = item.status='disputed' + dispute_note**。老師接受申訴(`handleDisputeResolve`)會**回頭改原卷 grading_result**（該題 isCorrect=true、score=maxScore、reason='申訴通過…'、從 mistakes 移除、重算分）。
- **endpoint = `correction_passed`**，目前會擋下 Phase A 重跑。
- **100 分 ≠ endpoint**：endpoint 只看 `correction_passed`，跟分數無關。

## 3. 核心原則

1. **學生交的東西永遠不清**：學生作答照片、訂正照片、申訴理由 = 事實，任何重跑都保留。
2. **從哪一格重來，就清那一格以下**：上游動作清得多、下游動作清得少。
3. **人為決定 AI 不能粗暴覆蓋**：老師已接受的申訴(`資料寫回原卷`)、進行中的申訴、已完成的訂正，重批時要按狀態矩陣（§5）謹慎處理。

## 4. 三個觸發 × 保留/清除

| 觸發 | 老師意圖 | 清除 | 保留 | endpoint 處理 |
|---|---|---|---|---|
| **改答案卷** | 「標準變了」 | grading_result、score、score_source、phase_a_state、final_answers、**correction_question_items、assignment_student_state(含 endpoint)** | 學生作答/訂正照片 | **force 清掉**（連 correction_passed 也清，回到 synced/graded） |
| **重跑 Phase A**（重新批改/重新截取） | 「AI 讀錯了」 | 同上但 **endpoint 擋下不清**（correction_passed 回 409） | 答案卷、學生照片 | **擋下**（不可重跑已完成的卷） |
| **重跑 Phase B**（只重算分） | 「讀對了、只是分數錯/我調了配分」 | 分數、mistakes（**逐題比對、見 §5**） | **phase_a_state、final_answers（不用重審）**、未變動題的訂正/申訴 | 不整批清；逐題處理 |

> ⚠️ **改答案卷 vs 重跑 Phase A 的關鍵差異**：改答案卷是「標準變了」，連已完成(endpoint)的卷都必須失效 → **force 清**；重跑 Phase A 只是重讀，已完成的卷不該被打擾 → **擋下**。

## 5. 重跑 Phase B 的逐題狀態矩陣（不整批清、只動「對錯翻轉」的題）

重批後逐題比對「新原始對錯 vs 舊原始對錯」，依該題目前狀態處理：

| 目前狀態 | 重批判「對」 | 重批判「錯」 |
|---|---|---|
| 對（非錯題） | 不動 | 變新錯題（item: open、未訂正） |
| 錯・未訂正(open) | 移除 mistake、給對、item→resolved/刪 | 不動 |
| 錯・訂正中(open, attempt 進行中) | 原判變對 → 給對、取消該題訂正（留學生已交紀錄） | 不動（讓學生繼續訂） |
| 錯・訂正完成(resolved) | 給對（訂正白做但不罰、紀錄保留） | **保留 resolved**（原判沒變、不用重訂） |
| 錯・申訴中(disputed) | **自動平反**：item→resolved、回寫原卷給對、**從老師待辦移除**、留「已自動通過」紀錄 | **保留 disputed**（爭議還在、交給人決定，不自動駁回） |
| 申訴成功(resolved, 老師已接受) | 不動（一致） | **保留**（老師決定 > AI、永不被覆蓋） |

**只有「原始對錯翻轉」的題會動到狀態；沒翻轉的題（含其訂正/申訴成果）一律保留。**

### 申訴自動平反細節
- 申訴中 + 重批判對 → 視為「系統自己承認原判錯」→ **自動判申訴成功**（沿用 `handleDisputeResolve` 的回寫邏輯：原卷該題 isCorrect=true/score=maxScore、移除 mistake、重算分）。
- **從老師「待處理申訴」清單移除**（老師不用再裁決），但**留一筆「申訴—因重新批改自動通過」紀錄**（學生看得到通過、老師事後可追溯）。
